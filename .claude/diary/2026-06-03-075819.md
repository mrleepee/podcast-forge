## Lessons Learned
- **Voice-clone echo bug (the big one).** OmniVoice's `generate(ref_audio=..., ref_text=None)` auto-transcribes AND trims/removes-silence on the reference, which misaligns audio↔text and makes the model **echo reference fragments into every chunk**. Our first reference clip was cut mid-flow ("...Creating a fresh") so a stray **"fresh"** leaked at the start of all 24 chunks of ep106. Fix: pass an **explicit, accurate `ref_text`** (disables the lossy trim — see `omnivoice/models/omnivoice.py` ~lines 47–49) and use a clean clip. Verified by transcribing each output segment. Stored ref_text in `voice_ref/senora_freedom_en_ref.txt` and auto-load it.
- **Short samples hide the echo.** The 1–2 chunk CLONED preview sounded fine; the bug only screamed on a long 24-chunk episode. Always test clone quality on a multi-chunk render, and transcribe the output to catch leaked words programmatically.
- **Subagent for a 2h GPU job is fragile.** The background re-render agent kept "completing" then **relaunching the killed render** on each wake — produced defective audio twice and ignored my kill. Had to `TaskStop` the agent + `pkill` processes. Takeaway: for long GPU batch jobs, own it as a background Bash job (PID-waiter), not a monitoring subagent. Also: a detached background process **survives** the agent that spawned it — pkill by name, don't assume the agent dying stops the work.
- **MPS is a single device** — fanning out parallel TTS subagents would contend/OOM (each model ~2.4GB on 16GB unified). Sequential, model-loaded-once is both faster and safe.
- **Throughput reality:** ~0.7× realtime at speed 1.0 (410s for a 4.7-min track), slower at 0.85. ~14 min/episode for EN+ES. The trial's "3.5× realtime" claim was optimistic. `num_step` (default 32 diffusion steps) is the speed lever if needed.
- OmniVoice output is 24kHz float32 mono — matches the pipeline's hardcoded `SAMPLE_RATE=24000`, no resampling. Masters cleanly to ≈−16 LUFS via existing `checks/master_audio.py`.
- `.gitignore` excludes `*.wav` — had to `git add -f` the reference clip (a source asset, not generated output) so the pipeline is self-contained.
