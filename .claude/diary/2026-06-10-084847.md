## Lessons Learned

- **The repo is busy/multi-agent.** Mid-session a scheduled cron fired and `git branch` showed I
  was on a *stale* branch (`feature/omnivoice-audio-quality-fix`) with my Phase 4 branch moved to
  an episode commit. I almost mis-read this as live concurrent modification; the user clarified
  "nothing happened when you were not working" — the other branches/worktrees (`checkin-worktree-*`,
  ep131/132/136) are just pre-existing state. Lesson: treat on-disk git state as ground truth,
  re-create branches cleanly from main rather than trusting the conversation's mental model.
- **`.claude/scheduled_tasks.json` churns constantly** — a background scheduler rewrites it, so
  every `git checkout`/`pull` tripped "local changes would be overwritten". Workaround that worked:
  `git stash push -- .claude/scheduled_tasks.json .claude/scheduled_tasks.lock` before each switch,
  then drop the stash. Left these files untracked per the user's pre-flight choice.
- **Fail-closed gates can break production if the asset they guard isn't ready.** The ref-clip
  hard-fail is correct per spec, but defaulting it ON would have halted the daily podcast because
  the clip re-cut is human work. Shipping the *capability* default-OFF + loud warning + a
  `voice_ref/README.md` is the honest middle ground.
- **Don't let strict gates nuke legacy data.** The publish gate + duplicate-number assertion both
  needed grandfathering (`publish_overrides.json`, `overridden` short-circuit) so the 124-episode
  live catalog wasn't retroactively pulled.
- **Test premises can be wrong because of shared state.** `test_empty_spoken_form_does_not_cover`
  initially used `tmux`, which is *also* in the real `pronunciation_cache.json` → covered there →
  test failed. Fixed by using a novel term (`frobnicate`). Similarly the seam-QC echo detector
  false-flagged at 1/3 occurrences (33% > 30%); fixed by requiring `count >= 2`.
- **`generate_rss` the module vs `generate_rss` the function** shadow each other — `import
  generate_rss; generate_rss(...)` calls the module. Must use `generate_rss.generate_rss(...)`.
- **pyright "could not be resolved" noise** for numpy/soundfile/kokoro/yt_dlp is just this venv's
  missing optional deps; not real errors. The `.venv` (py3.12) has pytest + yt_dlp; system py3.14
  has neither and is the right interpreter to prove the lazy-import fix.
