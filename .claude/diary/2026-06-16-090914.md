## Lessons learned (this session's failure-mode map)

- **"Both backends down simultaneously" is a red flag for a shared/local cause, not two outages.** The user caught this: this very chat + the Bash classifier + my pipeline's glm fallback all hit `api.z.ai`/glm Coding Plan — under load they strain *together*. MiniMax's "DNS failures" were transient local-resolver flakes; a `ping`/`gethostbyname` probe to multiple hosts instantly distinguished local-network vs service outage.
- **Killed batch runs orphan `tts_omnivoice.py` + `multiprocessing.resource_tracker` workers** that deadlock the next run (main thread blocks in `read`/`fcntl` on a pipe the dead worker never fills). `pkill -f _batch_produce_7` does NOT catch them (they're `python -c`/`tts_omnivoice`/`resource_tracker`). **Always `pkill -9 -f tts_omnivoice` + kill orphaned `multiprocessing.resource_tracker` before re-running.**
- **urllib's `timeout=N` is per-socket-op, not wall-clock** — it does NOT fire on slow-drip SSL reads (MiniMax/glm hung 10+ min with no exception). The fix is a daemon-thread + `join(hard_timeout)` + abandon pattern.
- **The quality gates are fail-closed and silent in non-interactive mode:** similarity (e.g. OKF flagged 28% vs ep108 — false positive), sponsorship (Hassabis flagged 7/10 as DeepMind promo), verification (13 untraceable claims on a fresh arxiv paper). `--force` bypasses similarity/sponsorship for vetted-distinct topics; `pipeline='summary'` avoids verification entirely.
- **The omni-voice Kokoro TTS is the real bottleneck under load**, not the LLM. A full episode (EN + 88-segment Spanish pass) takes ~18–22 min when system load is 5–13; it only completes reliably in low-load windows. `tts_omnivoice.py` itself has no multiprocessing — the deadlock is inside Kokoro's `KPipeline`.
- **zsh bracket-range globs lie:** `ls ep14[2-7]*` fails with "no matches" even when files exist. Use `find` or `grep` in a pipeline, not raw bracket globs, when checking produced artifacts.
- **`gh pr create` from the wrong branch returns the *existing* PR** for that branch, not a new one. Use `--head <branch> --base main` explicitly when creating a PR for a worktree-pushed branch.
