## Lessons Learned

### `pronunciation_cache.json` is the single source of truth
The pronunciation check relies entirely on this file (67 entries). When adding new episodes with terms like CBDC or x402, they must be seeded here. The `pronunciation_db.py` can auto-fetch from Wiktionary but common tech terms (tmux, CBDC) won't be there — `MANUAL_SEEDS` dict handles those.

### `produce_podcast()` doesn't write to `episodes.json`
The Python pipeline only generates audio + updates the vector index. The episodes.json description is written by the producing agent (the subagent in the podcast skill). This means the authoritative links feature is an agent instruction, not code — it works as long as the agent follows the skill prompt.

### Quality gate is soft, not hard
`_run_quality_gate()` in `video_downloader.py` warns on failure but doesn't block publish. This is intentional — the first iteration should prove itself before hardening. A future `--force-publish` flag pattern could be used to allow override when it becomes a hard gate.

### ffmpeg loudnorm is a single-pass fix
The two-pass approach (measure then normalise) works perfectly: ep102 went from -23.9 → -16.1 LUFS. The `master_audio.py` module already handles "already in tolerance" by skipping the second pass.

### Fixtures must be curated carefully
- The good fixture (`liberland-meritocracy.mp3`) is a real mastered episode — not raw Kokoro output
- The `tech-jargon.txt` fixture needed to use only terms from `pronunciation_cache.json` for the pronunciation check to pass
- Solo scripts naturally skip the dialogue check (PASS with "not applicable")
