## Lessons learned

- **The "Ai" bug was two layers:** missing registry titles (â†’ slug fallback) AND
  `.title()` acronym mangling. Fixing only the titles would recur; fixing both
  (PRs #40 + #41) is durable.
- **The feed is audio-driven, not registry-driven.** `find_podcast_episodes`
  globs the **audio dir**, then looks up metadata by slug. An episode with audio
  but no registry title â†’ slug-derived title. An episode in the registry with no
  audio â†’ silently absent from the feed.
- **Regenerating the feed re-keys unfrozen GUIDs.** `generate_rss` defaults an
  item's guid to `freeist:<slug>` when none is frozen in the registry â€” so old
  **URL-GUID** episodes (ep24/ep63) would change identity on regen â†’ duplicate in
  clients. **Always run `--snapshot-guids` first**, or do surgical edits.
- **Surgical > regenerate for live-feed title changes.** String-replace the title
  in `episodes.json`+`rss/feed.xml`+`feed.xml`; `git add` only those files. Avoids
  both GUID risk and `publish_feed()`'s `git add -A` sweeping in unrelated
  in-flight production audio.
- **Verify "live" claims against `origin/main`, not the local working tree.** I
  mis-stated "Hancock live 3Ã—" from local assumptions; `git show origin/main:â€¦`
  showed 0. Local freeist-podcast had dirty in-flight state (deleted/readded
  audio) that didn't reflect the published feed.
- **`publish_feed()`'s `git add -A` is a footgun** when the freeist-podcast tree
  is dirty (it had ~26 uncommitted audio files from a cron run). Never run it
  blindly; selective `git add` instead.
- **The cron/agent reads the backlog top-down, first non-struck item** â€” no
  automated parser. "Make it #1" = insert + renumber. Struck entries (~~â€¦~~) are
  the skip mechanism.
- **zsh globs lie in `git` pathspecs** (`ls ep14[2-7]*` / unquoted `*` â†’ "no
  matches"). Quote pathspecs (`'audio/ep147-*'`) or use `git ls-tree`/loops.
- **X.com posts fetch fine via webReader** (post text + metadata); **Instagram is
  login-walled** (garbled); **GitHub raw** via webReader works.
