  Why separate agents per stage is the right call:

  1. Smaller, sharper prompts — each agent does ONE thing well. The evidence
  agent doesn't need to know about hooks. The script agent doesn't need to
  extract claims.
  2. Testable artifacts at each stage — you can inspect the claim_ledger.json
  and outline.md before the script is written. Right now you can't see what
  MiniMax threw away between transcript and narration.
  3. Model flexibility — Stage 1 (extraction) could use a cheaper/faster model.
  Stage 3 (scripting) needs the best model. Right now everything uses one
  MiniMax call.
  4. The loop — if Content QA fails, you only re-run Stage 3 (script revision)
  with the QA feedback, not the whole pipeline. Currently you'd have to
  re-summarise and re-narrate from scratch.
  5. No information loss — the claim ledger persists. The script agent always
  has the evidence available, not a compressed summary.

  The implementation: Each stage is a Python function with its own prompt
  template, not a separate process. They all live in video_downloader.py (or a
  new pipeline.py) and run sequentially within produce_podcast(). The
  intermediate files (claim_ledger.json, outline.md) are saved to disk so you
  can inspect them.
